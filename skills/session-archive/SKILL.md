# Session Archive（對話歸檔）

這個 skill 將對話內容歸檔至 Vault，同時萃取對話中涉及的知識主題，
分別寫入知識筆記和對話筆記，形成完整的知識沉澱記錄。

Frontmatter Schema 說明見 `${CLAUDE_PLUGIN_ROOT}/skills/knowledge-archive/references/frontmatter-spec.md`。

---

## 4 步流程

### 步驟 1：內容獲取 Agent

**輸入**：session 來源（當前對話或指定歷史對話）

**執行**：
- 若為當前對話：獲取完整對話原文（含所有來回訊息）
- 若使用者指定歷史 session：讀取指定來源

**輸出**：完整對話原文

---

### 步驟 2：內容分析 Agent

**輸入**：步驟 1 的完整對話原文

**執行**：
- 產出 200 字以內的對話摘要
- 識別對話中涵蓋的多個獨立知識主題（通常 2-5 個）
- 為每個主題預擬知識筆記標題（中文，簡潔明確）
- 決定對話筆記的概述（10 字以內，用於檔名）
- 確認 session-id（對話識別碼，可用日期時間表示）

**輸出**：
```
摘要：[200 字以內]

主題列表：
1. [主題一標題]
2. [主題二標題]
...

對話概述：[10 字以內，用於檔名]
session-id：[對話識別碼]
```

---

### 步驟 3：知識筆記 Agent × N（平行執行）

等步驟 2 完成後，每個主題各啟動一個 Agent，平行處理，互不等待。

**輸入（各 Agent 獨立）**：
- 負責的主題名稱與預定標題
- 完整對話原文（供深入萃取）
- 整體摘要（供上下文理解）
- 對話筆記的預定檔名（供 source 欄位填寫，格式：`[序號]_[概述]`）

**執行**：
1. 針對該主題從對話中萃取重點，撰寫知識筆記正文
2. 正文中適當位置加入對話來源連結：`[[對話筆記檔名]]`
3. 呼叫 `tag-review` skill 取得標籤建議
4. 確認 `category`（等於 `tags[0]` 第一層）
5. 確定存放路徑：`主題知識/[YYYY-MM-DD]/[標題].md`
   - 日期為當日（格式 `YYYY-MM-DD`）
   - 子資料夾不存在時動態建立
6. 寫入筆記

**筆記格式**：

```markdown
---
title: [主題標題]
date: [YYYY-MM-DD]
tags:
  - [層級結構標籤，如 技術/AI/LLM]
  - [拆解標籤1]
  - [拆解標籤2]
  - [其他標籤...]
source: "[[對話筆記檔名]]"
category: [tags[0] 第一層]
content_type: conversation
author:
---

[主題相關的深度整理正文]

> 來源：[[對話筆記檔名]]
```

**輸出**：已寫入的筆記路徑與完整檔名（供步驟 4 使用）

---

### 步驟 4：對話筆記 Agent

**等步驟 3 全部完成後才執行**。

**輸入**：
- 對話摘要（來自步驟 2）
- 所有知識筆記的完整檔名列表（來自步驟 3）
- session-id、對話概述（來自步驟 2）

**執行**：
1. 確認今日在 `歷史紀錄/對話/[YYYY-MM-DD]/` 已有幾個對話筆記，決定序號（從 01 開始）
2. 確定檔名：`[序號]_[概述].md`（如 `01_AI架構討論.md`）
3. 正文包含摘要，並以 wikilink 列出所有相關知識筆記
4. 寫入 `歷史紀錄/對話/[YYYY-MM-DD]/[序號]_[概述].md`
   - 日期子資料夾不存在時動態建立

**筆記格式**：

```markdown
---
title: [對話標題]
date: [YYYY-MM-DD]
source: [session-id]
category: 對話歷史
content_type: ClaudeCode
author: [使用者名稱或留空]
---

## 摘要

[200 字以內的對話摘要]

## 相關知識筆記

- [[知識筆記一標題]]
- [[知識筆記二標題]]
```

**輸出**：已寫入的對話筆記路徑

---

```
- [[主題二標題]] → 主題知識/[date]/[標題].md
...
```
