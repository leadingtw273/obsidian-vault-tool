# 主題領域層標籤規格（Tag Topic Spec）

Agent 在執行 Tag Review 時，必須依照本文件的階層結構與組合規則生成標籤。

---

## 標籤組合規則

每篇筆記的 `tags` 欄位由三段組成，總數不超過 **10 個**：

```
[層級結構標籤] + [層級拆解標籤] + [其他標籤]
```

| 段落 | 說明 | 數量 |
|------|------|------|
| **層級結構標籤** | 完整分層路徑，格式 `一級/二級/三級` | 1 個 |
| **層級拆解標籤** | 將層級結構的每一節展開為個別平坦標籤 | 2–3 個 |
| **其他標籤** | 筆記內容的具體關鍵詞、應用場景、工具名稱等 | 2–5 個 |

### 範例

一篇關於 LLM Prompt 技巧的筆記：

```yaml
tags:
  - 技術/AI/LLM           # 層級結構標籤
  - 技術                   # 層級拆解
  - AI                     # 層級拆解
  - LLM                    # 層級拆解
  - 提示詞工程              # 其他
  - 思維鏈                  # 其他
```

## 標籤命名規則

| 規則 | 說明 |
|------|------|
| **語言** | 中文為主；通俗英文縮寫保留原文（AI、LLM、Agent、Python、TypeScript） |
| **層級分隔符** | `/`，Obsidian 原生支援分層 |
| **拆解展開** | 層級結構標籤的每一節直接作為平坦標籤，原樣展開 |
| **其他標籤** | 具體關鍵詞、工具名、應用場景；以中文為主，通俗英文縮寫可保留原文 |
| **禁止混入** | `category`、`status` 等 frontmatter 屬性欄位的值不得出現在 tags |

## 判斷流程（Agent 使用）

```
1. 分析筆記內容，判斷屬於哪個一級領域
   → 技術 / 商業 / 知識 / 學習 / 個人 / 研究

2. 在該領域中，找最精確的子分類（二級或三級）
   → 選出唯一的層級結構標籤（如 技術/AI/LLM）

3. 將該層級結構標籤的每一節展開為平坦標籤
   → 技術/AI/LLM → 技術、AI、LLM

4. 根據筆記具體內容補充其他標籤
   → 工具、技術術語、應用場景、人名等，2–5 個

5. 驗證總數 ≤ 10 個
```
